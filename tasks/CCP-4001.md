# CCP-4001: Introduce AccessUser resource

## Description

AccessUser represents a developer or system user who can access site
files. It is separate from the ContainerCP User resource.

## Requirements

Create:

libs/access/
    AccessUser.h
    AccessUser.cpp
    AccessUserManager.h
    AccessUserManager.cpp

Namespace:

containercp::access

### AccessUser struct (inherits Resource)

```
- username       (unique per site)
- password       (future: hashed; store plaintext for now)
- site_id        (foreign key to Site)
- domain         (denormalized, for display)
- provider       (string: "sftp", "ftp", "webdav")
- path           (filesystem path)
- enabled        (bool)
```

### AccessUserManager

```
- create(username, site_id, domain, provider, path)
- remove(id)
- find(username)
- find(id)
- list()
- list_by_site(site_id)
- set_users(...)
```

### Storage

Add `access_users.db`:

```
id|username|site_id|domain|provider|path|enabled
```

### Registration

- `ServiceRegistry` exposes `access_users()`
- No default seeding needed

## Acceptance criteria

1. AccessUser record can be created and persisted
2. AccessUser is linked to a Site
3. All existing commands still work
