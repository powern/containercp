# CCP-3001: Add test framework

## Description

The project has no test framework. Every verification is manual.
Add doctest (single-header) as the test framework.

## Requirements

### Download or create

If doctest is available in the system or can be downloaded:

- Place the single header in `libs/doctest/doctest.h`
- Create `tests/CMakeLists.txt` and a test executable target
- Add `find_package` or embed the header directly

### CMake integration

Add to the root `CMakeLists.txt`:

```cmake
option(BUILD_TESTS "Build tests" ON)
if(BUILD_TESTS)
    enable_testing()
    add_subdirectory(tests)
endif()
```

### Test executable

Create `tests/main.cpp` and `tests/runner.cpp`:

- `#define DOCTEST_CONFIG_IMPLEMENT` in `runner.cpp`
- `#define DOCTEST_CONFIG_IMPLEMENTATION` in `main.cpp`
- Register all test files

### Build and run

```
cmake --build build
ctest --output-on-failure
```

## Acceptance criteria

1. `ctest` runs and reports "0 failures"
2. Test executable compiles without errors
3. Framework does not pollute production code
