# CCP-4006: Document future SFTP implementation

## Description

Create a design document for the future SFTP implementation.
No code changes — only documentation.

## Requirements

Create:

docs/SFTP-PROVIDER.md

### Document contents

**Chroot strategy:**
- Each SFTP user is chrooted to their site path
- Use OpenSSH internal-sftp with `ChrootDirectory`
- Site path owned by root, writeable subdirectory inside

**Permissions model:**
- Site files owned by a dedicated system user per site
- Access users mapped via group membership
- Read/write per access user configurable in future

**SSH key authentication:**
- Public keys stored per AccessUser
- SSH authorized_keys file generated per user
- Only key auth allowed for SFTP users

**Password authentication:**
- Disabled by default for SFTP users
- Future: optional password auth with rate limiting

**No classic FTP by default:**
- FTP is plaintext and insecure
- Explicit FTPS (FTP over TLS) considered optional
- Recommended: SFTP only

**Optional FTP provider:**
- Pure-FTPd or vsftpd container
- Separate FTP port per site
- Configurable enable/disable
- FTPS mandatory if enabled

## Acceptance criteria

1. `docs/SFTP-PROVIDER.md` exists
2. Document covers chroot, permissions, keys, passwords, FTP
3. All existing commands still work
