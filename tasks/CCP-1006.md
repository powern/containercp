# CCP-1006: SSL resource

## Description

SSL certificate metadata must be tracked as a resource. Let's Encrypt
integration is deferred. The resource establishes where certificates
live and how they map to domains.

## Requirements

Create:

libs/ssl/
    SslCertificate.h
    SslCertificate.cpp
    SslManager.h
    SslManager.cpp

### SslCertificate struct (inherits Resource)

```
- domain_id (foreign key to Domain)
- certificate_path (e.g., ./ssl/fullchain.pem)
- private_key_path (e.g., ./ssl/privkey.pem)
- expires_at (Unix timestamp, 0 = unknown)
- auto_renew (bool, default true)
- node_id
```

### SslManager

```
- create(domain_id)
- remove(id)
- list_by_domain(domain_id)
- set_certificates(...)
```

### Storage

Add `ssl_certificates.db`:

```
id|name|domain_id|certificate_path|private_key_path|expires_at|auto_renew|node_id
```

### Registration

- `ServiceRegistry` exposes `ssl()`
- `SiteLayout` already creates `ssl/` directory — this resource
  formalises its purpose

## Acceptance criteria

1. SSL certificate record can be created
2. Certificate is linked to a Domain
3. All existing commands still work
