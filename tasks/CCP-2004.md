# CCP-2004: Implement site remove

## Description

ContainerCP must be able to completely remove a site and all of its
resources.

## Requirements

CLI command:

```
containercp site remove <domain>
```

### What must be removed

1. Docker Compose stack: `docker compose down` via Runtime
2. Site directory: `/srv/containercp/sites/<domain>/`
3. All linked Domain resources (find by site_id)
4. All linked Database resources (find by site_id)
5. All linked Backup resources (find by site_id)
6. All SSL certificates for the site's domains
7. All Mail domains for the site's domains
8. The Site record itself

### Implementation

- Add a new operation: `SiteRemoveOperation`
- Use existing `Runtime::remove_site()` for compose down
- Use `Filesystem::remove_directory()` for filesystem cleanup
- Iterate through each manager and remove linked resources
- Remove the site record last
- Save after successful removal

### Safety

- Require confirmation if `--force` is not passed:

```
WARNING: This will permanently remove example.com and all its data.
Are you sure? [y/N]
```

- Print summary of what was removed

## Acceptance criteria

1. `site remove example.com` removes everything
2. Non-existent domain returns error
3. `site list` no longer shows the removed site
4. All existing commands still work
