# CCP-2003: Implement username validation

## Description

`containercp user create` and `containercp site create` must validate
the username / owner parameter before creating any resources.

## Requirements

Add to `Validator`:

```
static bool is_valid_username(const std::string& username);
```

Validation rules:

- Length 3–32 characters
- Only: A-Z, a-z, 0-9, underscore, hyphen
- Cannot start with a digit or hyphen
- Cannot end with a hyphen or underscore

### Integration

- Call `Validator::is_valid_username()` in:
  - `handle_user_create()` before creating the user
  - `SiteCreateOperation::execute()` before creating the site

- Return error: "Invalid username: <reason>" if validation fails

## Acceptance criteria

1. `admin` passes
2. `john_doe` passes
3. `ab` fails (too short)
4. `1admin` fails (starts with digit)
5. `admin-` fails (ends with hyphen)
6. All existing commands still work
