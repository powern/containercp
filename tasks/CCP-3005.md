# CCP-3005: Add --force confirmation flow for site remove

## Description

`containercp site remove <domain>` currently removes everything
without asking for confirmation. This is dangerous for a hosting
control panel.

## Requirements

### Behavior

When `--force` is NOT passed:

```
WARNING: This will permanently remove example.com and all its data.
Are you sure? [y/N]
```

If the user does not type 'y' or 'yes', abort with exit code 1.

When `--force` IS passed:

```
containercp site remove example.com --force
```

Remove without prompting.

### CLI parsing

Update the CLI handler to check for `--force` as the third argument:

```cpp
if (argc == 4 && arg1 == "site" && argv[2] == "remove") {
    // no --force
}
if (argc == 5 && arg1 == "site" && argv[2] == "remove" && argv[4] == "--force") {
    // force mode
}
```

### Implementation notes

- Use `std::getline(std::cin, line)` for user input
- Accept "y", "Y", "yes", "YES" as confirmation
- Any other response aborts

## Acceptance criteria

1. `site remove domain` (without `--force`) prompts before removing
2. Cancelling the prompt aborts the operation
3. `site remove domain --force` removes without prompting
4. `site remove non-existent --force` returns "Site not found"
5. All existing commands still work
