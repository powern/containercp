# CCP-3002: Add unit tests for Validator

## Description

The `Validator` class provides `is_valid_hostname()` and
`is_valid_username()`. These must have comprehensive unit tests.

## Requirements

Create `tests/test_validator.cpp` with test cases:

### Hostname tests

- `""` — empty (invalid)
- `"a"` — single label without dot (invalid)
- `"example.com"` — valid
- `"sub.domain.local"` — multi-label valid
- `"-bad.com"` — starts with hyphen (invalid)
- `"bad-.com"` — ends with hyphen (invalid)
- `"a..b"` — empty label (invalid)
- `"toolonglabel-aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa.com"` — label > 63 chars (invalid)
- 254-char hostname (invalid, over 253)

### Username tests

- `""` — empty (invalid)
- `"ab"` — too short (invalid)
- `"admin"` — valid
- `"john_doe"` — underscore valid
- `"1admin"` — starts with digit (invalid)
- `"-admin"` — starts with hyphen (invalid)
- `"admin-"` — ends with hyphen (invalid)
- `"admin_"` — ends with underscore (invalid)
- `"BadUser"` — uppercase (invalid)
- `"user name"` — space (invalid)

## Acceptance criteria

1. All hostname tests pass
2. All username tests pass
3. `ctest` reports 0 failures
