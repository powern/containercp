# CCP-4004: Add CLI commands for access user management

## Description

Full CRUD for access users via CLI.

## Requirements

### Commands

```
containercp access user create <username> <domain>
containercp access user list
containercp access user show <username>
containercp access user disable <username>
containercp access user enable <username>
containercp access user remove <username>
```

### Behavior

**create:**
- Look up the domain in DomainManager, get site_id
- Generate path: `/srv/containercp/sites/<domain>/public`
- Set provider to "sftp" (default)
- Generate random password (via PasswordGenerator)
- Create AccessUser resource
- Call AccessProvider::create_user()
- Display the generated password

**list:**
- Iterate AccessUserManager::list()
- Print: id, username, domain, enabled

**show:**
- Find by username
- Print all fields except password

**disable / enable:**
- Find by username
- Set enabled flag
- Call provider::disable_user() or enable_user()
- Save

**remove:**
- Find by username
- Call provider::remove_user()
- Remove resource
- Save

### Help text

Add to print_help():

```
  access user create <username> <domain> Create access user
  access user list                       List access users
  access user show <username>            Show access user
  access user disable <username>         Disable access user
  access user enable <username>          Enable access user
  access user remove <username>          Remove access user
```

## Acceptance criteria

1. All six commands parse correctly
2. `access user create` creates a resource and prints the password
3. `access user list` shows created users
4. `access user disable` sets enabled to false
5. `access user enable` sets enabled to true
6. `access user remove` removes the resource
7. All existing commands still work
