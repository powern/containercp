# CCP-4005: Generate per-site access directory mapping

## Description

Access users must be linked to the correct site path. The path is
determined at create time and stored in the AccessUser resource.

## Requirements

### Path resolution

When creating an access user for domain `example.com`:

```
path = /srv/containercp/sites/example.com/public
```

This is the `public/` directory within the site layout.

### Validation steps

1. Verify the domain exists via DomainManager
2. Verify the site exists (site_id from domain)
3. If the `public/` directory does not exist on disk, log a warning
   but do not fail (site may not have been created yet)

### Storage

The path is stored in `access_users.db` and reloaded on startup.

## Acceptance criteria

1. Created access users have the correct path
2. Path references the site's public directory
3. Non-existent site returns an error
4. All existing commands still work
