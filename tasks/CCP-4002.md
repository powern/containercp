# CCP-4002: Introduce AccessProvider abstraction

## Description

Access providers must be pluggable. Create an abstract interface
that SFTP, FTP, and WebDAV providers can implement.

## Requirements

Create:

libs/access/
    AccessProvider.h
    AccessProvider.cpp

Namespace:

containercp::access

### AccessProvider interface

```
class AccessProvider {
public:
    virtual ~AccessProvider() = default;

    virtual core::OperationResult create_user(const AccessUser& user) = 0;
    virtual core::OperationResult remove_user(const AccessUser& user) = 0;
    virtual core::OperationResult disable_user(const AccessUser& user) = 0;
    virtual core::OperationResult enable_user(const AccessUser& user) = 0;
    virtual core::OperationResult status(const AccessUser& user) = 0;
};
```

### Registration

- ServiceRegistry holds a concrete provider instance
- Default provider: `LocalSftpProvider`
- Future: switch provider via Config or CLI flag

## Acceptance criteria

1. AccessProvider interface compiles
2. ServiceRegistry exposes `access_provider()`
3. All existing commands still work
