# CCP-1001: User resource

## Description

ContainerCP must model hosting users as a first-class resource,
similar to how VestaCP manages users with packages.

## Requirements

Create:

libs/user/
    User.h
    User.cpp
    UserManager.h
    UserManager.cpp

### User struct (inherits Resource)

```
- username (unique)
- email
- package (e.g., "default", "premium")
- node_id
```

### UserManager

```
- create(username, email, package)
- remove(id)
- find(username)
- list()
- set_users(...)
```

### Storage

Add `users.db` with pipe-delimited format:

```
id|username|email|package|node_id
```

### Registration

- `ServiceRegistry` exposes `users()`
- Application seeds a default "admin" user on first run

## Acceptance criteria

1. User is persisted across restarts
2. Default admin user exists after first run
3. All existing commands still work
