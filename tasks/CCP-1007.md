# CCP-1007: Mail placeholder resource

## Description

Mail is explicitly out of scope for the MVP, but the resource model
must reserve the concept so that CLI commands and Storage are ready
when mail support is implemented in a future sprint.

## Requirements

Create:

libs/mail/
    MailAccount.h
    MailAccount.cpp
    MailManager.h
    MailManager.cpp

### MailAccount struct (inherits Resource)

```
- email (full email address)
- mailbox_type (enum: "local", "forward")
- forward_to (empty for local mailboxes)
- site_id
- node_id
```

### MailManager

```
- create(email, mailbox_type, site_id)
- remove(id)
- list_by_site(site_id)
- set_accounts(...)
```

### Storage

Add `mail_accounts.db`:

```
id|name|email|mailbox_type|forward_to|site_id|node_id
```

### Registration

- `ServiceRegistry` exposes `mail()`

## Acceptance criteria

1. Mail account record can be created
2. Mail account is linked to a Site
3. All existing commands still work
