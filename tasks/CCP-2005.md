# CCP-2005: Improve .env generation

## Description

The `.env` file currently contains hardcoded `site_db` and `site_user`
values. Database resource credentials must be used instead.

## Requirements

### EnvGenerator changes

- `EnvGenerator::generate()` must accept database credentials:

```
EnvGenerator(filesystem::Filesystem& fs, const std::string& site_dir);
bool generate(const std::string& domain, const std::string& owner,
              const std::string& db_name, const std::string& db_user,
              const std::string& db_password);
```

- Use the passed values for `DB_NAME`, `DB_USER`, `DB_PASSWORD`

### DockerComposeProvider changes

- After creating the database resource, retrieve its credentials
- Pass them to `EnvGenerator::generate()`

### Backward compatibility

- Old `.env` files without DB_NAME etc. remain valid
- Only newly created sites get per-site database credentials

## Acceptance criteria

1. New `.env` files contain the database credentials from the resource
2. `DB_NAME` matches the Database resource's `db_name`
3. `DB_USER` matches the Database resource's `db_user`
4. `DB_PASSWORD` matches the Database resource's `db_password`
5. All existing commands still work
