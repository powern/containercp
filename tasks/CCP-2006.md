# CCP-2006: Implement dry-run mode

## Description

`containercp site create` must support `--dry-run` to validate
parameters without writing anything to disk.

## Requirements

CLI usage:

```
containercp site create admin demo.local --dry-run
```

### Behavior

When `--dry-run` is passed:

1. Validate all parameters (domain, username)
2. Check that the site does not already exist
3. Check that Docker is available
4. Print what would be created
5. Do NOT create any of the following:
   - Site record in SiteManager
   - Domain record in DomainManager
   - Database record in DatabaseManager
   - Site directory on disk
   - docker-compose.yml
   - .env file
   - README.txt
   - nginx config
   - index.php
   - Any Docker commands

### Output example

```
[Dry Run] Would create site: demo.local
[Dry Run] Would create domain: demo.local
[Dry Run] Would create database: demo_local_db
[Dry Run] Would generate docker-compose.yml
[Dry Run] Would create directory: /srv/containercp/sites/demo.local/
[Dry Run] Would start Docker stack
```

### Integration

- Add a `dry_run` parameter to `SiteCreateOperation::execute()`
- The operation checks the flag and skips all mutation

## Acceptance criteria

1. `--dry-run` with valid parameters prints plan and exits 0
2. `--dry-run` with invalid domain prints error and exits 1
3. No filesystem changes are made
4. All existing commands still work
