# CCP-3006: Improve validation error messages

## Description

Validation currently returns generic messages like "Invalid domain."
and "Invalid username." without explaining what was wrong. This
makes debugging hard for users.

## Requirements

### Validator changes

Change the validator functions to return a reason string instead of
a boolean:

```cpp
// Returns empty string on success, error message on failure
static std::string validate_hostname(const std::string& hostname);
static std::string validate_username(const std::string& username);
```

Keep the original boolean functions for backward compatibility or
replace them entirely.

### Error messages

**Hostname:**
- "Domain is empty"
- "Domain is longer than 253 characters"
- "Label is longer than 63 characters: &lt;label&gt;"
- "Label cannot start with a hyphen: &lt;label&gt;"
- "Label cannot end with a hyphen: &lt;label&gt;"
- "Label contains invalid character: &lt;char&gt;"
- "Domain must contain at least one dot"

**Username:**
- "Username is empty"
- "Username must be between 3 and 32 characters"
- "Username cannot start with a digit"
- "Username cannot start with a hyphen"
- "Username cannot end with a hyphen"
- "Username cannot end with an underscore"
- "Username can only contain lowercase letters, digits, hyphens, and underscores"
- "Username cannot contain uppercase letters"

### Integration

Update all callers to use the new detailed validation and pass the
specific error message through `OperationResult`.

## Acceptance criteria

1. `site create admin ""` returns "Domain is empty"
2. `site create admin -bad.com` returns "Label cannot start with a hyphen: -bad"
3. `user create BAD` returns "Username cannot contain uppercase letters"
4. All existing commands still work
