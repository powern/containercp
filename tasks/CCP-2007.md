# CCP-2007: Cache Docker availability

## Description

`DockerRuntime::check_docker()` calls `std::system()` on every
command invocation. This is wasteful — Docker availability does not
change during a single process lifetime.

## Requirements

### DockerRuntime changes

- Cache the result of the first `check_docker()` call in a boolean
  member variable
- Return the cached result on subsequent calls
- Reset cache if needed (future use)

```
class DockerRuntime : public Runtime {
private:
    bool docker_checked_ = false;
    bool docker_available_ = false;
};
```

### Performance

- First call: runs `docker --version` as before
- Subsequent calls: return cached value, no system call

## Acceptance criteria

1. First Docker command triggers the check
2. Subsequent commands use the cached result
3. `std::system("docker --version")` is called only once per process
4. All existing commands still work
