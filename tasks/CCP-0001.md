# CCP-0001: Generate complete site directory layout

## Description

When `containercp site create` runs, the provider must create a full
directory tree for the site instead of just the compose file.

## Required directories

```
/srv/containercp/sites/<domain>/
├── www/          # web root
├── public/       # public entry point (linked or sub)
├── logs/         # nginx + php error logs
├── tmp/          # php temporary files
├── ssl/          # TLS certificates (placeholder)
└── backups/      # future backup storage
```

## Acceptance criteria

1. All directories exist after `site create`
2. Directories are writable by the container user
3. Existing directories do not cause errors
4. README.txt is still created at the site root
5. docker-compose.yml references correct volume paths
