# CCP-2002: Implement domain validation

## Description

`containercp site create` must validate the domain parameter before
creating any resources. Invalid domains must be rejected with a clear
error message.

## Requirements

Create:

libs/utils/
    Validator.h
    Validator.cpp

### DomainValidator

```
static bool is_valid_hostname(const std::string& hostname);
static std::string normalize(const std::string& hostname);
```

Validation rules (RFC 1123 + practical):

- Total length ≤ 253 characters
- Each label ≤ 63 characters
- Labels contain only: A-Z, a-z, 0-9, hyphen
- Labels cannot start or end with a hyphen
- Must contain at least one dot (e.g., example.com, not just "local")

### Integration

- Call `Validator::is_valid_hostname()` in `SiteCreateOperation::execute()`
  before any resource creation
- Return error: "Invalid domain: <reason>" if validation fails

## Acceptance criteria

1. `example.com` passes
2. `demo.test.local` passes
3. `-bad.com` fails (starts with hyphen)
4. `bad-.com` fails (ends with hyphen)
5. `a` fails (no dot)
6. All existing commands still work
