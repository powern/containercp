# CCP-4003: Implement LocalSftpProvider placeholder

## Description

LocalSftpProvider implements AccessProvider but does NOT configure
real SFTP yet. It logs actions and returns success.

## Requirements

Create:

libs/access/
    LocalSftpProvider.h
    LocalSftpProvider.cpp

### Behavior

Each method logs via Logger:

```
[INFO] LocalSftpProvider: Creating user <username> for <domain>
[INFO] LocalSftpProvider: Removing user <username>
```

Return `OperationResult{true, ""}` for every method.

No Linux user creation.

No sshd_config changes.

No chroot setup.

### Registration

- `DockerComposeProvider`-style: wired in `ServiceRegistry`
- ServiceRegistry creates `LocalSftpProvider` and exposes it as
  `AccessProvider&`

## Acceptance criteria

1. LocalSftpProvider compiles without warnings
2. Provider methods do nothing but log
3. All existing commands still work
