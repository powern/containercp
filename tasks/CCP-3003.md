# CCP-3003: Add unit tests for managers and Storage

## Description

Managers and Storage are the core data layer. They must have tests
to ensure create/find/remove/list round-trips work correctly.

## Requirements

Create `tests/test_managers.cpp` and `tests/test_storage.cpp`.

### Manager tests (UserManager, SiteManager)

- Create a resource, find by name, verify fields
- List returns the created resource
- Remove by id, find returns nullptr
- Remove non-existent returns false
- Create with duplicate name still works (unique constraint not enforced)

### Storage round-trip tests

Use a temporary directory for storage files.

- Save User, load User, verify all fields
- Save Site, load Site, verify all fields (including new db fields)
- Save Domain, load Domain, verify all fields
- Load from non-existent file returns empty vector
- Empty vector save produces empty file

## Acceptance criteria

1. All manager tests pass
2. All Storage round-trip tests pass
3. Temporary files are cleaned up after tests
4. `ctest` reports 0 failures
