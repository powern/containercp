# CCP-1004: Database resource

## Description

Databases must be tracked as independent resources linked to a Site.
This allows listing, creating, and removing databases separately from
the compose stack lifecycle.

## Requirements

Create:

libs/database/
    Database.h
    Database.cpp
    DatabaseManager.h
    DatabaseManager.cpp

### Database struct (inherits Resource)

```
- db_name
- db_user
- db_password (stored in memory only, generated at create time)
- db_type (e.g., "mariadb", "postgresql")
- site_id
- node_id
```

### DatabaseManager

```
- create(db_name, db_user, db_type, site_id)
- remove(id)
- find(id)
- list_by_site(site_id)
- set_databases(...)
```

### Storage

Add `databases.db`:

```
id|name|db_name|db_user|db_type|site_id|node_id
```

### Registration

- `ServiceRegistry` exposes `databases()`
- `EnvGenerator` generates credentials for each database
- DB credentials are added to the site's `.env` file

## Acceptance criteria

1. Database record is persisted
2. Database is linked to a Site
3. Credentials appear in `.env`
4. All existing commands still work
