# CCP-1002: Domain resource

## Description

Domains must be tracked as separate resources linked to a Site.
This enables future DNS and SSL features that operate on domains
independently of the site record.

## Requirements

Create:

libs/domain/
    Domain.h
    Domain.cpp
    DomainManager.h
    DomainManager.cpp

### Domain struct (inherits Resource)

```
- domain (the domain name)
- site_id (foreign key to Site)
- node_id
```

### DomainManager

```
- create(domain, site_id)
- remove(id)
- find(domain)
- list()
- list_by_site(site_id)
- set_domains(...)
```

### Storage

Add `domains.db`:

```
id|name|domain|site_id|node_id
```

### Registration

- `ServiceRegistry` exposes `domains()`
- `SiteCreateOperation` automatically creates the primary domain
  when a site is created

## Acceptance criteria

1. Creating a site auto-creates its primary domain
2. Domain is persisted across restarts
3. All existing commands still work
