# CCP-1005: Backup resource

## Description

Backup metadata must be tracked as a resource. Actual backup
execution is deferred to a future sprint. The resource establishes
the data model so CLI commands can be added later.

## Requirements

Create:

libs/backup/
    Backup.h
    Backup.cpp
    BackupManager.h
    BackupManager.cpp

### Backup struct (inherits Resource)

```
- site_id
- backup_type (enum: "full", "database", "files")
- status (enum: "pending", "completed", "failed")
- file_path
- size_bytes
- node_id
```

### BackupManager

```
- create(site_id, type)
- remove(id)
- list_by_site(site_id)
- set_backups(...)
```

### Storage

Add `backups.db`:

```
id|name|site_id|backup_type|status|file_path|size_bytes|node_id
```

### Registration

- `ServiceRegistry` exposes `backups()`

## Acceptance criteria

1. Backup record can be created and stored
2. Backup is linked to a Site
3. All existing commands still work
