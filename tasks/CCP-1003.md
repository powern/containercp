# CCP-1003: PHP version abstraction

## Description

PHP version must be configurable per site, not hardcoded in the
compose template. Classic panels let users choose between PHP
versions for each site.

## Requirements

Create:

libs/php/
    PhpVersion.h
    PhpVersion.cpp
    PhpSettings.h
    PhpSettings.cpp

### PhpVersion enum

```
enum class PhpVersion {
    v8_3,
    v8_4,
};
```

### PhpSettings struct

```
- version (PhpVersion)
- memory_limit (string, default "256M")
- upload_limit (string, default "128M")
- site_id
```

### Compose integration

- `ComposeGenerator` must accept a `PhpVersion` parameter
- The generated compose file uses the correct PHP image tag
  (`php:8.3-fpm` or `php:8.4-fpm`)
- `Site` gains a `php_version` field

### Storage

Add `php_settings.db`:

```
id|site_id|version|memory_limit|upload_limit
```

## Acceptance criteria

1. Site stores its PHP version
2. Compose file reflects the chosen PHP version
3. Default is `v8_4`
4. All existing commands still work
