# Mail Module Implementation Plan

## Project Goal

Make ContainerCP a production-grade mail platform with automated
integration testing against real DNS domains and email services.

## Test Environment

Three DNS-isolated mail domains on `web2.softico.ua` (116.202.231.94):

| Domain | MX | Purpose |
|--------|----|---------|
| maillab.softi.co | maillab.softi.co | Primary mail server (SMTP, IMAP, aliases, DKIM) |
| relay.softi.co | relay.softi.co | External relay testing (smarthost, auth relay) |
| mailhub.softi.co | mailhub.softi.co | Split delivery, M365 integration, hybrid routing |

All domains have A, MX, SPF, DMARC records. DKIM to be generated by ContainerCP.

## Architecture

```
API layer (ApiServer.cpp)
    │
    ▼
MailDomainManager / MailboxManager / MailAliasManager
    │
    ▼
DockerMailProvider (Postfix + Dovecot + Redis)
    │
    ├── apply_config() → generate → validate → reload → rollback
    ├── check_health() → per-service + per-process checks
    ├── activate/deactivate lifecycle
    └── RuntimeSynchronizer integration
```

### Known Problems
1. Remote test server has old daemon binary — needs rebuild + deploy
2. No integration tests run against real DNS domains yet
3. Dovecot config validation not possible (`doveconf -n` not in container)
4. `crypt()` not thread-safe (`crypt_r` needed)
5. `relay_host` not format-validated
6. Port bindings hardcoded (not configurable)
7. No `POST /api/mail/recover` endpoint
8. No MailHealthMonitor periodic checks

### Risks
- Dovecot may fail on fresh install if SSL cert generation has issues
- Postfix check inside container requires container to be running
- DKIM key generation requires OpenSSL in container
- Remote server may have port conflicts (25, 143, 465, 587, 993)

## Rollback Strategy

1. `apply_config()` backs up current config in memory before generation
2. If `postfix check` fails → restore from backup
3. If Postfix reload fails → restore from backup
4. Stale staging directories cleaned up after apply
5. Previous known-good config preserved in generated/ until next successful apply

## Deployment

```bash
# 1. Build on dev machine
cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release
cmake --build build-release

# 2. Copy to test server
scp build-release/containercpd ai-agent@maillab.softi.co:/tmp/

# 3. Deploy on test server
ssh ai-agent@maillab.softi.co
sudo systemctl stop containercpd
sudo cp /tmp/containercpd /usr/local/bin/containercpd
sudo systemctl start containercpd

# 4. Verify
curl http://127.0.0.1:8080/api/version
```
